# ISOMAC - IT Asset Management System

A full-stack inventory and asset management application built with React, Node.js, Express, and MongoDB.

## 🎯 Overview

ISOMAC is a comprehensive asset management system designed to help organizations track IT assets, manage employee assignments, handle maintenance schedules, and generate reports. This is a personal project showcasing modern web development practices.

## Quick Start

Prerequisites:
- Node.js 16+ and npm (or yarn)

Clone the repository and install dependencies from the project root:

```
git clone <repository-url>
cd <repo-root>
npm install
```

For development, follow the root README or the specific package README for services you plan to run (backend, frontend, scripts).

## Git & Commit Guidelines

- Create feature branches from `main`: `git checkout -b feat/short-description`
- Keep commits small and focused. Use imperative present tense in messages (e.g., "Add asset export endpoint").
- Include issue or ticket IDs in the commit message when applicable.
- Rebase or squash local commits before opening a pull request to keep history clean.

Example commit message:

```
feat(assets): add CSV export for inventory reports

Closes: ISSUE-123
```

## Pull Request Checklist

- Target branch: `main` (or the appropriate release branch)
- Provide a clear description and screenshots or sample output when relevant
- Add tests for new features or fixes
- Ensure linting and unit tests pass locally

## Contributing

1. Fork the repository (or create a branch if you have direct push access).
2. Implement changes on a descriptive branch name.
3. Run tests and linters.
4. Open a Pull Request and request reviewers.

## Code Style & Hooks
